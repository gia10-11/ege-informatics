Тип 26 № 27423

Условие: Системный администратор раз в неделю создаёт архив пользовательских файлов. Однако объём диска, куда он помещает архив, может быть меньше, чем суммарный объём архивируемых файлов. Известно, какой объём занимает файл каждого пользователя.

По заданной информации об объёме файлов пользователей и свободном объёме на архивном диске определите максимальное число пользователей, чьи файлы можно сохранить в архиве, а также максимальный размер имеющегося файла, который может быть сохранён в архиве, при условии, что сохранены файлы максимально возможного числа пользователей.

Входные данные: В первой строке входного файла находятся два числа: S — размер свободного места на диске (натуральное число, не превышающее 10 000) и N — количество пользователей (натуральное число, не превышающее 1000). В следующих N строках находятся значения объёмов файлов каждого пользователя (натуральные числа, не превышающие 100).

Пример входного файла:
100 4
80
30
50
40

При таких исходных данных можно сохранить файлы максимум двух пользователей. Возможные объёмы этих двух файлов — 30 и 40, 30 и 50 или 40 и 50. Наибольший объём файла из перечисленных пар — 50, поэтому ответ для приведённого примера: 2 50.

📁 Скачать файл: 26_demo.txt

Решение: Сначала считываем данные из файла. После этого сортируем массив в порядке возрастания. Таким образом, последовательно складывая элементы массива с начала и сравнивая сумму с размером свободного места на диске, получим максимальное количество пользователей, чьи файлы могут поместиться на диске. Далее, вычитая из найденной суммы наибольший файл в текущей последовательности, будем пробовать прибавлять файлы с большим весом. Если такой файл будет найден, то заменяем значение наибольшего файла, который возможно поместить на диск.

Алгоритм:
1. Сортируем файлы по возрастанию
2. Берём наименьшие файлы, пока они помещаются
3. Находим остаток места
4. Ищем максимальный файл, который можно добавить вместо последнего взятого

Ответ: 568 50

# Решение на Python
f = open('26_demo.txt')
data = f.readlines()
s = int(data[0].split()[0])
del data[0]
for i in range(len(data)):
    data[i] = int(data[i])
data.sort()
summa = 0
for count in range(len(data)):
    if summa + data[count] > s:
        break
    summa += data[count]
print(count)
zapas = s - summa
for i in range(len(data)):
    if data[i] - data[count - 1] <= zapas:
        itog = data[i]
print(itog)

По заданной информации об объёме файлов пользователей и свободном объёме на архивном диске определите максимальное число пользователей, чьи файлы можно сохранить в архиве, а также максимальный размер имеющегося файла, который может быть сохранён в архиве, при условии, что сохранены файлы максимально возможного числа пользователей.

Входные данные: В первой строке входного файла находятся два числа: S — размер свободного места на диске (натуральное число, не превышающее 10 000) и N — количество пользователей (натуральное число, не превышающее 4000). В следующих N строках находятся значения объёмов файлов каждого пользователя.

📁 Скачать файл: 27880.txt

Решение: Аналогично заданию 1. Сортируем файлы, берём наименьшие, пока хватает места, затем пытаемся заменить последний взятый файл на максимально возможный из оставшихся.

Алгоритм:
1. Считываем файлы в список
2. Сортируем список по возрастанию
3. Последовательно добавляем файлы, пока сумма ≤ S
4. Находим максимальный файл, который можно добавить вместо последнего

Ответ: 611 27

f = open('27880.txt')
s, n = map(int, f.readline().split())
v = sorted(map(int, f))
summa, cnt = 0, 0
for i in range(len(v)):
    if summa + v[i] <= s:
        summa += v[i]
        cnt += 1
biggest_file = v[cnt - 1] + s - summa
while biggest_file not in v:
    biggest_file -= 1
print(cnt, biggest_file)

Условие: Системный администратор раз в неделю создаёт архив пользовательских файлов. Однако объём диска, куда он помещает архив, может быть меньше, чем суммарный объём архивируемых файлов. Известно, какой объём занимает файл каждого пользователя.

По заданной информации об объёме файлов пользователей и свободном объёме на архивном диске определите максимальное число пользователей, чьи файлы можно сохранить в архиве, а также максимальный размер имеющегося файла, который может быть сохранён в архиве, при условии, что сохранены файлы максимально возможного числа пользователей.

Входные данные: В первой строке: S (≤ 10000) и N (≤ 3000).

📁 Скачать файл: 28139.txt

Решение: Аналогично. Сортируем файлы, набираем максимальное количество, затем пробуем заменять файлы на более тяжёлые.

Ответ: 200 10

f = open('28139.txt')
s, n = map(int, f.readline().split())
v = sorted(map(int, f))
summa, cnt = 0, 0
for i in range(len(v)):
    if summa + v[i] > s:
        break
    summa += v[i]
    cnt = i + 1
t = v[cnt-1]
for i in range(cnt, len(v)):
    if summa - t + v[i] <= s:
        summa = summa - t + v[i]
        t = v[i]
print(cnt, t)

Условие: Продавец предоставляет покупателю, делающему большую закупку, скидку по следующим правилам:

— на каждый второй товар стоимостью больше 50 рублей предоставляется скидка 25%;
— общая стоимость покупки со скидкой округляется вверх до целого числа рублей;
— порядок товаров в списке определяет продавец и делает это так, чтобы общая сумма скидки была наименьшей.

По известной стоимости каждого товара в покупке необходимо определить общую стоимость покупки с учётом скидки и стоимость самого дорогого товара, на который будет предоставлена скидка.

Пример входного файла:
6
125
50
490
215
144
320
В данном случае товар стоимостью 50 не участвует в определении скидки, остальные товары продавцу выгодно расположить в таком порядке цен: 490, 125, 215, 144, 320. Тогда скидка предоставляется на товары стоимостью 125 и 144. Стоимость этих двух товаров со скидкой составит 201,75 рублей, после округления — 202 рубля. Общая стоимость покупки составит: 50 + 490 + 215 + 320 + 202 = 1277 рублей. Самый дорогой товар, на который будет получена скидка, стоит 144 рубля. В ответе нужно записать числа 1277 и 144.

📁 Скачать файл: inf_22_10_20_26.txt

Решение: Заметим, что числа, меньшие 51, можно сразу суммировать, поскольку на них скидка не распространяется. Поэтому, построчно считывая числа из файла, числа, меньшие 51, будем сразу накапливать в переменной sum, а остальные числа будем записывать в массив. Далее, отсортировав массив по возрастанию, будем прибавлять к переменной sum стоимость товара с учётом скидки, если данный элемент массива имеет индекс, меньший, чем количество чисел, поделённое пополам, и без учёта скидки в остальных случаях.

Алгоритм:
1. Сортируем все товары дороже 50 рублей
2. На каждый второй (самый дешёвый) товар даём скидку 25%
3. Округляем общую сумму вверх

Ответ: 469784 511

import math
f = open('inf_22_10_20_26.txt')
a = []
m = int(f.readline())
s = 0
for i in f:
    x = int(i)
    if x <= 50:
        s += x
    else:
        a.append(x)
a.sort()
for i in range(len(a)):
    if i < len(a) // 2:
        s += a[i] * 0.75
        maxi = a[i]
    else:
        s += a[i]
print(math.ceil(s), maxi)

Пример входного файла:
6 605

120
160
100
340
В данном случае сначала нужно взять груз массой 205 кг. После этого можно вывезти ещё максимум 3 груза. Выбираем способ: 140 + 160 + 100. Всего получается 4 груза общей массой 605 кг. В ответе надо записать числа 4 и 605.

📁 Скачать файл: 26.txt

Решение: Сначала берём все грузы из диапазона 200-210 кг. Затем сортируем остальные грузы по возрастанию и берём наименьшие, пока есть место. После этого пытаемся заменить последние взятые грузы на более тяжёлые для максимизации весов.

Ответ: 123 10000

f = open('26.txt')
n, m = map(int, f.readline().split())
g0 = [int(s) for s in f]
g = [x for x in g0 if not (200 <= x <= 210)]
mass = sum(g0) - sum(g)
k = len(g0) - len(g)
g.sort()
kuzov = []
while mass + sum(kuzov) + g[0] <= m:
    kuzov.append(g.pop(0))
for i in range(len(kuzov)-1, -1, -1):
    limit = m - mass - sum(kuzov) + kuzov[i]
    gruz = [x for x in g if x <= limit]
    if not gruz:
        break
    gr = max(gruz)
    g.append(kuzov[i])
    kuzov[i] = gr
    g.remove(gr)
print(k + len(kuzov), mass + sum(kuzov))

f = open('26 (1).txt')
_, money = map(int, f.readline().split())
items = []

    t = line.split()
    items.append([t[2], int(t[0]), int(t[1])])

zapB = 0

    if tip == 'B':
        zapB += buy
    money -= buy * price

Алгоритм:
1. Разделяем числа на чётные и нечётные
